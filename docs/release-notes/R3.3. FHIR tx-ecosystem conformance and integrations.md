# Release Notes: R3.3 — FHIR Terminology Ecosystem Conformance & Integrations

**Version:** 3.3.0 (in development)
**Period covered:** 15 May 2026 – 9 July 2026
**Type:** Major Feature Release

---

## Abstract

R3.3 is a **FHIR terminology conformance** release. TermX's `$expand`, `$validate-code`, `$lookup`,
`$translate` and `$batch-validate-code` operations were brought in line with the official
**HL7 FHIR Terminology Ecosystem (tx-ecosystem)** test suite: version-aware resolution, hierarchical
expansion, first-class **CodeSystem supplements**, richer designation handling and structured
`OperationOutcome` responses — all exercised by a built-in **conformance test runner and UI**. The
release also unifies the **ValueSet expansion engine** (strict per-version with a parity gate), adds
the **Phase P integrations** (wiki PDF/HTML export, Azure DevOps git sync, terminology-server OAuth2
with secret encryption, data-driven global search, skins), extends **SNOMED/LOINC import** (object-store
streaming, delta generation, national editions), and ships a **StructureDefinition UML** view.

This note lists **new capabilities only**; the many conformance-driven defect fixes in the same window
are folded into the capability summaries rather than listed individually.

---

## For business users

- **Standards-conformant terminology server.** TermX now answers the FHIR terminology operations the
  way the HL7 ecosystem expects — the same behaviour clients get from reference servers such as
  tx.fhir.org. That means **fewer surprises when integrating** external systems, validators and IGs.
  A built-in **conformance dashboard** shows exactly which official test cases pass.
- **Publish documentation from the wiki.** Any wiki page — or a whole space — can be **exported to PDF
  or HTML**, ordered by the space's menu, for sharing with stakeholders who don't log in to TermX.
- **Work with content in Azure DevOps.** Alongside the existing GitHub workflow, TermX can now
  **synchronise terminology and modelling content with Azure DevOps repositories**, so teams already on
  Azure DevOps keep their existing source-control home.
- **Connect to secured terminology servers.** External terminology servers can be reached with
  **OAuth2 / API-key / basic auth**, and their stored secrets are **encrypted at rest**.
- **Faster, safer big imports.** SNOMED and LOINC imports stream from an object store, support
  **delta analysis** and **national editions**, and **email you when a long import finishes or fails**.
- **Richer modelling views.** StructureDefinitions gain a **UML diagram tab**, and code-system/value-set
  versions sort by **semantic version (latest first)** everywhere.
- **Tailored look and quicker navigation.** Deployments can apply a **skin** (e.g. a cs-gov theme),
  the **global search** dashboard is data-driven per space, and new **keyboard shortcuts** speed up
  everyday work.

## For deployment / operations

- **Version:** targets **3.3.0** (the `VERSION` file is bumped at release cut).
- **FHIR conformance tooling (server image).** The `termx-app` image bakes in the HL7 validator jar for
  the conformance runner:
  - `TERMX_CONFORMANCE_VALIDATOR_JAR` — path to `validator_cli.jar` (baked into the image;
    `VALIDATOR_CLI_URL` build-arg pins the release).
  - `TERMX_CONFORMANCE_TX_URL` — the FHIR base this server exposes as the `-tx` target of a run
    (default `http://localhost:8200/fhir`; override per deployment).
  - See [`fhir-tx-conformance-todo.md`](../fhir-tx-conformance-todo.md) and the conformance feature notes.
- **Azure DevOps sync (opt-in).** Configure `ms-devops.client.*` (PAT-based). The bean is **opt-in** —
  endpoints degrade gracefully when unconfigured.
- **Terminology-server secret encryption (opt-in).** `termx.server.secret-encryption-key` enables
  AES/GCM encryption of stored server secrets; **plaintext passthrough** when unset (no migration needed
  to adopt later).
- **External terminology-server auth.** Server auth config gains an **auth type**
  (`none` / `basic` / `oauth2` / `apikey`) and `scope`, dispatched centrally by the HTTP client.
- **Per-space global search.** A per-space **`global_search`** flag (DB, changeset
  `space-global-search`) drives the global-search dashboard instead of a hardcoded env list.
- **kefhir dependency** advances to **R5.12**, resolved **exclusively from GitHub Packages** — ensure
  `GITHUB_ACTOR` / `GITHUB_TOKEN` (or the packages token) are available to the build.
- **Object store / Snowstorm** remain required for the streaming SNOMED/LOINC import paths
  (`BOB_MINIO_*`, `SNOWSTORM_*`).
- **DevOps note:** the new config keys above are being folded into
  [`deployment-configuration.md`](../features/deployment-configuration.md).

## Technical details

### FHIR Terminology Ecosystem conformance
- **Conformance test runner** driving the official `tx-ecosystem` suite via `validator_cli.jar txTests`,
  emitting a FHIR `TestReport`; **loadSetup** auto-loads setup content (`#208`, `#217`, `#303`).
- **Conformance UI** (Phase 4) for browsing suite results (web `#100`).
- **`$expand`**: hierarchical (nested `expansion.contains`), version negotiation on inline expand,
  `property`/`designation` parameters honoured, supplement designations grafted, used-codesystem /
  used-supplement parameters, `activeOnly` / tri-state `compose.inactive`, abstract/notSelectable
  member flags, cross-version resolution.
- **`$validate-code`** (CodeSystem & ValueSet): version-aware resolution, structured `OperationOutcome`
  issues with form-aware locations, display-language scoping, supplement-aware display validation,
  inactive/abstract handling, CodeableConcept multi-coding validation, `$batch-validate-code`.
- **`$lookup`**: standard properties (inactive/parent/child), `property=*` wildcard, supplement
  designations with a source part.
- **`$translate`**: translation against inline (tx-resource) ConceptMaps.
- **Inline `tx-resource` support**: `$expand` / `$validate-code` resolve `url` against value sets and
  code systems supplied in the request bundle.
- **CapabilityStatement / TerminologyCapabilities** metadata satisfying tx-ecosystem checks (`#367`).

### ValueSet expansion engine
- **Expansion unification** — strict per-version semantics with an **engine-parity gate** across the
  stored-snapshot and inline expansion functions (`#370`; see
  [`value-set-expand.md`](../features/value-set-expand.md)).
- Implicit value sets over a code system, expansion **total without `contains`**, paging/filter, and
  SNOMED implicit VS support.

### Phase P integrations
- **Wiki PDF/HTML export** — new `wiki-pdf` module, content ordered by `pages.json`; renders with
  Flying Saucer + OpenPDF (`#371`, `#372`).
- **Azure DevOps git sync** — `org.termx.core.msdevops` (PAT-based `AzureDevOpsClient` +
  `AzureToGithubMapper`) reusing the GitHub sync abstractions, per-domain data handlers (`#371`).
- **Terminology-server OAuth2 + secret encryption** — `TerminologyServerAuthConfig` auth types & scope,
  `ScopedOAuthTokenClient`, `SecretEncryptor` (AES/GCM) (`#371`).
- **Data-driven global search** — per-space `global_search` flag (`#371`, web `#107`).
- Frontend Phase P — wiki export, Azure DevOps, server OAuth2, **cs-gov skin / skin switching**,
  file-import templates (web `#103`, `#107`).

### SNOMED / LOINC import
- **Object-store (BOB/MinIO) streaming** import pipeline (phases 1a–1d), streaming scan-and-proceed.
- **SNOMED delta generator** (phase 2) + archive **file statistics**, **archive detail pages**,
  archives **scoped by code system**, **national editions**, **effective-time tracking**, and
  **import lifecycle email notifications**.
- **LOINC** import from stored archive, **version autodetect from filename**, archive-file overrides.

### Modelling, platform & UX
- **StructureDefinition UML** diagram tab via an external `fhir2uml` converter (web `#97`).
- **FHIR ETag revalidation** filter (`#163`, docs [`fhir-etag-revalidation.md`](../features/fhir-etag-revalidation.md)).
- **Build/info metadata** endpoint and info page (`#175`, web `#90`).
- **Property value extension** display & editing (web `#87`, `#89`).
- **Coding refresh** UI and `$summary`-aware FHIR list links (web `#62`, `#109`).
- **Semantic version sorting** for resource versions, backfilled version algorithm.
- **UCUM**: Estonian supplement + value set with alias resolution, fragment content model
  (`#374`, `#382`).
- Frontend **quick wins** — keyboard shortcuts, global-search, `translateList` (web `#105`).
