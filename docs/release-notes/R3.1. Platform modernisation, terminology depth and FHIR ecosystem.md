# Release Notes: R3.1 — Platform Modernisation, Terminology Depth & FHIR Ecosystem

**Version:** 3.1.0
**Period covered:** 1 January 2025 – 24 March 2026
**Type:** Major Release

---

## Abstract

R3.1 is the large consolidation release that carried TermX from early-2025 hardening into a
modernised early-2026 platform. It bundles three tracks: **platform modernisation** (Java 25,
package/API refactor, build & release pipeline, container-registry publishing, Sonar/SpotBugs
quality gates), **terminology depth** (FHIR CodeSystem **supplements** with supplement-aware
`$lookup`, **ValueSet rule-expansion preview**, inline expansion, designation pattern search,
validate-by-name, UCUM/FHIR-UCUM), and the flagship **FHIR Terminology Ecosystem** integration
(HL7 coordination-registry discovery/resolution plus a public web UI). The web and server tracks
converge in early 2026 around UCUM, supplement-based localisation, task/permission flows, and
value-set UX.

This note lists **new capabilities only**; defect fixes shipped in the same window are omitted.

---

## For business users

- **Discover and resolve terminology servers.** A new **FHIR Terminology Ecosystem** view lets you
  find which terminology servers across the HL7 ecosystem host a given code system, filter by FHIR
  version, and identify authoritative servers — through a no-login web page or a REST API.
- **Localise terminologies without duplicating them.** Full **CodeSystem supplement** support adds
  translations and extra properties on top of a base code system (e.g. national UCUM display names)
  without copying concepts; `$lookup` merges the supplement data automatically by language or on
  explicit request.
- **Preview value-set rules before saving.** The value-set rule editor can **expand a single include
  rule** (including SNOMED) and show the result before you commit a version — clearer dynamic-vs-static
  editing.
- **Know who sees what.** **Role-based task visibility** (Admin / Publisher / Editor / Viewer) means
  view-only users don't see the task list, editors see their own/assigned work, and publishers see the
  broader sets for resources they publish.
- **Get told when long imports finish.** **Email notifications** report completion and failures for
  major imports (SNOMED/Snowstorm, LOINC, file imports) so you don't have to watch a progress bar.
- **Work faster on large content.** Performance work on hierarchical browsing, exports and memory keeps
  large or "wide" classifications (lab nomenclatures, big lab code systems) responsive.
- **Move data in and out.** CSV/XLSX export and round-trip editing, plus richer cross-navigation between
  CodeSystem / ValueSet / ConceptMap with version and concept-status context.

## For deployment / operations

- **Version:** server and web release as **3.1.0**.
- **Runtime modernisation:** the backend moves to **Java 25**; container images are published to the
  **GitHub Container Registry**. Plan a full rebuild/redeploy of both server and web.
- **FHIR Terminology Ecosystem** (optional): works out of the box against HL7's public coordination
  server. To point at a custom registry set `TERMINOLOGY_ECOSYSTEM_URL`
  (`terminology-ecosystem.coordination-server-url`, default `http://tx.fhir.org/tx-reg`). The `/tx-reg`
  discovery/resolution API and the `/tx-ecosystem/` UI are public (no login). See
  [fhir-terminology-ecosystem-registry-proxy.md](../features/fhir-terminology-ecosystem-registry-proxy.md).
- **Email notifications** (optional): require SMTP — configure `SMTP_*` and the import recipients
  (`SMTP_TO_IMPORT`). See [smtp-email-support.md](../features/smtp-email-support.md) and
  [email-import-notifications.md](../features/email-import-notifications.md).
- **Configurable endpoints:** hardcoded URLs were replaced with configuration, so environments/endpoints
  are set via config rather than baked-in strings. Full reference:
  [deployment-configuration.md](../features/deployment-configuration.md).
- **Quality gates:** the release build runs Sonar and SpotBugs static analysis.

## Technical details

### FHIR Terminology Ecosystem integration
- **Discovery** `GET /tx-reg` (list registered servers) and **resolution** `GET /tx-reg/resolve`
  (find the authoritative server for a system/URL), proxying the HL7 coordination registry —
  standards-compliant with the FHIR Terminology Ecosystem IG and drop-in compatible with tx.fhir.org.
- **Web UI** at `/tx-ecosystem/` — discovery/resolution tabs, FHIR-version filter, JSON download,
  publicly accessible.
- Configurable coordination server (`terminology-ecosystem.coordination-server-url`); JSON responses
  with optional file download. Full reference:
  [fhir-terminology-ecosystem-registry-proxy.md](../features/fhir-terminology-ecosystem-registry-proxy.md).

### Terminology & FHIR
- **CodeSystem supplements** — first-class FHIR supplements (`content = supplement`) attaching
  designations/properties to a base system; **`$lookup`** merges supplement data with `useSupplement`
  (explicit) or `displayLanguage` (auto-discovery). See
  [code-system-supplement-and-lookup.md](../features/code-system-supplement-and-lookup.md).
- **ValueSet rule-expansion preview** — `POST /ts/value-sets/expand-rule` expands one include rule
  without persisting a version, feeding the rule-editor preview. See
  [value-set-rule-expansion-preview.md](../features/value-set-rule-expansion-preview.md).
- **Inline value-set expansion**, **designation pattern/regex search** (beyond exact match),
  **validate-by-name** / value-set validate code+display, and operation-definition contributions.
- **UCUM / FHIR-UCUM** implementation and units handling on FHIR paths.

### Platform, build & quality
- **Java 25** adoption; package rename and API refactor; build speedups and a reworked release
  pipeline publishing images to **GHCR**.
- **Sonar** and **SpotBugs** quality gates wired into CI (release + weekly).

### Import / export, access & performance
- **Email import notifications** (HTML, multi-recipient) across LOINC / SNOMED / file imports, on the
  SMTP stack. See [email-import-notifications.md](../features/email-import-notifications.md).
- **Role-based task access control** (Admin/Publisher/Editor/Viewer) and the TaskForge task model.
  See [task-access-control.md](../features/task-access-control.md) and
  [task-management.md](../features/task-management.md).
- **Performance/scale:** memory optimisation, hierarchical-browser and code-system list performance,
  faster CSV/XLSX export paths.
- **Configuration hygiene:** hardcoded URLs replaced with configurable endpoints.

### termx-web (frontend)
- **Readonly / detailed configuration views** with consistent metadata display.
- **UCUM** UI, **concept-reference widget**, and **rule-expansion** UI surfacing the server capabilities.
- **Koodivaramu** merge line (Estonian national code-system context) and routine upstream syncs.
- **Tree-view** and code-system list performance; large-diff viewing thresholds raised.
