# Release Notes: R3.2 — SNOMED RF2 Redesign & Terminology Depth

**Version:** 3.2.0
**Period covered:** 24 March 2026 – 14 May 2026
**Type:** Feature Release

---

## Abstract

R3.2 reworks how large **SNOMED CT RF2** archives are brought into TermX — a scan-first,
streaming pipeline with dry-run analysis, explicit import modes and phase-by-phase progress —
and deepens everyday terminology work: **filtered hierarchical browsing**, **persistent import
mappings**, a reworked **privileges & GitHub integration model**, **ConceptMap closure**, and
**"latest version" resolution** for code systems. The release also modernises the frontend stack
(Angular 21 + web-commons) and adds list-performance and CodeSystem-supplement improvements.

This note lists **new capabilities only**; defect fixes shipped in the same window are omitted.

---

## For business users

- **Predictable SNOMED imports.** Before committing a multi-gigabyte RF2 release, you can now run
  a **dry-run scan** that reports what the archive contains and what will change, choose an explicit
  **import mode**, and watch **live phase-by-phase progress** instead of a single opaque spinner.
  Long-running imports report their outcome without you having to keep the tab open.
- **Faster navigation of big code systems.** Hierarchical concept trees can be **filtered in place**,
  so you can find a concept inside a large classification without losing the tree context. Code system
  and value set **lists load noticeably faster**.
- **Less repetitive import setup.** Column-to-field **import mappings can be saved** and reused across
  imports of the same source, removing manual re-mapping every time.
- **Clearer governance.** The **privileges model** and the **GitHub-backed content workflow** were
  reworked so that access rights and repository-synced resources follow one consistent naming model.
- **Always work from the right version.** Code systems now resolve to their **latest applicable
  version** where that matters, and value-set rules show the **code, display and status** of a
  selected concept pattern directly in the rules table.

## For deployment / operations

- **Version:** server and web both bump to **3.2.0**.
- **Frontend platform bump:** `termx-web` moves to **Angular 21** on the shared **web-commons**
  component library — a full frontend rebuild/redeploy is required (no config change).
- **New / relevant configuration** (see
  [`deployment-configuration.md`](../features/deployment-configuration.md) for the full reference):
  - `GUEST_DISABLED` (server) / `GUEST_DISABLED` (web) — block unauthenticated/guest access.
  - `OAUTH_JWKS_CACHE_TTL_SECONDS` — JWKS cache TTL is now configurable.
  - GitHub token env-var name is now flexible (the integration accepts alternative token variable
    names), easing CI/hosting setups.
- **SNOMED streaming imports** use the object store (MinIO/`BOB_MINIO_*`) and Snowstorm
  (`SNOWSTORM_*`); if you enable the redesigned import path, ensure those are configured. See
  [`snomed-import-management.md`](../features/snomed-import-management.md) and
  [`snomed-rf2-dry-run-scan.md`](../features/snomed-rf2-dry-run-scan.md).
- **Database:** includes the StructureDefinition versioning migration
  (`11-structure_definition_versioning.sql`); Liquibase runs on startup as usual.

## Technical details

### SNOMED CT RF2 import redesign
- **Dry-run scan** of an RF2 archive: file statistics and a pre-import analysis before any write
  (`feature/snomed-rf2-dry-run-scan`, docs [`snomed-rf2-dry-run-scan.md`](../features/snomed-rf2-dry-run-scan.md)).
- **Import mode summary** and **progress phases** surfaced to the UI (`feature/snomed-rf2-mode-summary`,
  `feature/snomed-rf2-progress-phases`).
- **Progress reporting** via the long-running-operation (lorque) queue
  (`feature/lorque-progress-reporting` / web `feature/lorque-progress-display`).

### Terminology & FHIR
- **ConceptMap closure** computation and **on-demand concept closure**
  (`feature/conceptmap-closure`, `refactor/concept-closure-on-demand`).
- **Latest CodeSystem version** resolution (`Feature: latest codesystem changes`).
- **CodeSystem property rule bindings**: FHIR endpoint returning `codesystem.property` rule-binding
  info; property **coding display shown on `$lookup`**.
- **CodeSystem supplement** improvements and removal of UCUM leftovers (`#93`).
- **StructureDefinition**: relations model, refactor, and versioning migration.
- CodeSystem/ValueSet **import routed through the FHIR mapper** for consistency.

### UX & platform
- **Filtered hierarchical tree view** for concepts (server `#92` / web `#48`), including filter
  support in the CodeSystem hierarchical view.
- **Persistent import mappings** (web `#47`).
- **Privileges & GitHub content model** — Phase A and Phase B rename
  (`feat/privileges-github-model-phase-a`, `-phase-b-rename`).
- **List performance**: value set list and code system (TS API) list improvements.
- Ability to **disable the guest session provider** (`GUEST_DISABLED`).
- **Angular 21 + web-commons** migration (web `Switch-to-web-commons-and-Angular21`).
- Value-set **rule editor** shows selected pattern's code / display / status in the rules table.
