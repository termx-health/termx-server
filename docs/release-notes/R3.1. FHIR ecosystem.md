## 🎉 What's New

### FHIR Terminology Ecosystem Integration

TermX now includes full integration with the HL7 FHIR Terminology Ecosystem, providing both a programmatic API and an interactive web interface for discovering and resolving terminology servers across distributed healthcare systems.

## ✨ Key Features

### 🌐 Interactive Web Interface (TermX)
- **New UI** at `/tx-ecosystem/` on TermX for visual exploration of terminology servers (HL7 coordination registry)
- Modern, responsive design that works on desktop, tablet, and mobile
- Two-tab interface: Discovery and Resolution
- Download results as JSON files
- No login required - publicly accessible

### 🔌 RESTful API (TermX — coordination proxy)
- **Discovery endpoint** (`/tx-reg`) for listing all registered terminology servers
- **Resolution endpoint** (`/tx-reg/resolve`) for finding authoritative servers
- Standards-compliant with HL7 FHIR Terminology Ecosystem IG
- Same API as HL7's tx.fhir.org - drop-in compatible
- JSON responses with optional file download
- **Note:** These paths are served by **TermX** (e.g. dev.termx.org), not by **HTX Router** demos such as tx.hl7.lt (router uses `/fhir`, `/ecosystem`, Swagger—see below).

### 🎯 Core Capabilities
- Discover all terminology servers in the HL7 ecosystem
- Filter by FHIR version (R3, R4, R4B, R5, R6)
- Find servers hosting specific code systems (SNOMED CT, LOINC, etc.)
- Identify authoritative vs. candidate servers
- View server access methods (open, OAuth, token authentication)
- Export server information for integration or documentation

### HTX platform (operational gateway + browser)

TermX **tx-reg** / **tx-ecosystem** integrate the **HL7 coordination** registry (discovery and resolution). Many deployments also run a **terminology gateway** and **browser** for **local** servers using the same [FHIR Terminology Ecosystem IG](https://build.fhir.org/ig/HL7/fhir-tx-ecosystem-ig/ecosystem.html) patterns—see **[Feature description](../features/fhir-terminology-ecosystem-feature-description.md)** (“Related ecosystem deployments: HTX Router and HTX Viewer”).

**Important:** The public demo **HTX Router** at **tx.hl7.lt** is **not** a TermX server: it does **not** expose TermX’s **`/tx-reg`** or **`/tx-ecosystem/`** coordination API. Those endpoints are provided by **TermX** (see examples below). On tx.hl7.lt you get the **router’s** FHIR API, **`GET /ecosystem`**, Swagger UI, etc.

**Example demo sites**

- **HTX Router (gateway / API):** https://tx.hl7.lt — [OpenAPI / Swagger UI](https://tx.hl7.lt/swagger-ui/index.html)  
- **HTX Viewer (web UI):** https://htx.hl7.ee  

## 🚀 Getting Started

### TermX — coordination registry (HL7 `tx-reg` proxy)

Use a **TermX** deployment for **`/tx-reg`** discovery/resolution and **`/tx-ecosystem/`** UI (not tx.hl7.lt).

#### For End Users - Web Interface

Navigate to:
```
https://dev.termx.org/tx-ecosystem/
```

**Example Workflow:**
1. Open the web UI
2. Select FHIR Version "R4"
3. Click "Discover Servers"
4. Browse available terminology servers
5. Click "Download Results" to export

#### For Developers - TermX API

**Find all R4 servers:**
```bash
curl https://dev.termx.org/tx-reg?fhirVersion=R4
```

**Find SNOMED CT server:**
```bash
curl "https://dev.termx.org/tx-reg/resolve?fhirVersion=R4&url=http://snomed.info/sct"
```

**Download server list:**
```bash
curl "https://dev.termx.org/tx-reg?download=true" -o servers.json
```

### HTX Router at tx.hl7.lt (gateway — no `tx-reg`)

Use these for the **HTX Router** demo. Paths match [Swagger UI](https://tx.hl7.lt/swagger-ui/index.html).

**Ecosystem summary (sanitized config + resource counts):**
```bash
curl -sS https://tx.hl7.lt/ecosystem | jq .
```

**Terminology capabilities (all code systems known to the router):**
```bash
curl -sS "https://tx.hl7.lt/fhir/metadata?mode=terminology"
```

**FHIR base and docs in browser:**
- Router landing: https://tx.hl7.lt/  
- FHIR API index: https://tx.hl7.lt/fhir/  
- **Swagger UI:** https://tx.hl7.lt/swagger-ui/index.html  

Typical terminology operations are under **`/fhir`** (e.g. CodeSystem `$lookup`, ValueSet `$expand`), as listed in Swagger—not under `/tx-reg`.

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/tx-reg` | Discovery: list all servers (HL7 coordination registry) |
| GET | `/tx-reg/resolve` | Resolution: find server for terminology |
| GET | `/tx-ecosystem/` | TermX web UI for discovery / resolution |

### API Endpoints — HTX Router (e.g. tx.hl7.lt)

These are **not** `tx-reg`; see [Swagger](https://tx.hl7.lt/swagger-ui/index.html).

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/ecosystem` | Ecosystem summary (servers, counts, operations) |
| GET | `/fhir/metadata?mode=terminology` | TerminologyCapabilities |
| — | `/fhir/...` | FHIR terminology resources and operations (`$lookup`, `$expand`, …) |
| GET | `/swagger-ui/index.html` | OpenAPI / Swagger UI |

## ⚙️ Configuration

### Default Configuration
The feature works immediately with no configuration required. It uses the HL7 public coordination server by default.

### Optional Customization
To use a custom coordination server:

**application.yml:**
```yaml
terminology-ecosystem:
  coordination-server-url: https://custom.server/tx-reg
```

**Environment Variable:**
```bash
export TERMINOLOGY_ECOSYSTEM_URL=https://custom.server/tx-reg
```

## ⚠️ Known Limitations

### Current Version
1. **No Caching:** Every request queries the coordination server
   - Future enhancement planned for optional caching

2. **Single Coordination Server:** Queries one server at a time
   - Configurable via environment variable
   - Multi-registry support planned for future release

3. **Requires Coordination Server:** Returns 502 if coordination server unavailable
   - HL7 server has high uptime (99.9%+)
   - Local cache planned for future release

### Browser Requirements (Web UI)
- Modern browser required (Chrome, Firefox, Safari, Edge 2020+)
- JavaScript must be enabled
- Internet Explorer 11 not supported

## 🛠️ Troubleshooting

### Common Issues

**404 on `https://tx.hl7.lt/tx-reg` (or `/tx-ecosystem/`):** Expected. **tx.hl7.lt** runs **HTX Router**, not TermX. There is no **`/tx-reg`** there. Use **TermX** for coordination (`/tx-reg`, `/tx-ecosystem/`), or use the router’s **`/ecosystem`**, **`/fhir/...`**, and [Swagger](https://tx.hl7.lt/swagger-ui/index.html) for the gateway API.

**UI returns 404:**
- Verify application is running
- Check static resource configuration in application.yml

**API returns 502 (Bad Gateway):**
- Coordination server may be temporarily unavailable
- Try again in a few moments
- Check coordination server URL configuration

**Empty results:**
- Adjust filter criteria
- Try broader search parameters
- Verify coordination server is responding

**Required parameter error:**
- Ensure `fhirVersion` is provided for resolution endpoint
- Provide either `url` or `valueSet` parameter
